# ConsumerParameterBuilder

Builder class used to build custom parameters.

Consumer parameters are key/value pairs which can be provided during service consumption.
Official OceanProtocol [Consumer Parameters documentation](https://docs.oceanprotocol.com/developers/compute-to-data/compute-options#consumer-parameters)

## Usage
```ts twoslash
import { ConsumerParameterBuilder, ServiceBuilder, ServiceTypes, FileTypes } from '@deltadao/nautilus'

const consumerParameterBuilder = new ConsumerParameterBuilder() // [!code focus]
const numberParameter = consumerParameterBuilder // [!code focus]
    .setType('number') // [!code focus]
    .setName('numberEntries') // [!code focus]
    .setLabel('Number of Entries') // [!code focus]
    .setDescription('Set the number of entries used for computation.') // [!code focus]
    .setDefault(12) // [!code focus]
    .setRequired(false) // [!code focus]
    .build() // [!code focus]

const builder = new ServiceBuilder({ serviceType: ServiceTypes.ACCESS, fileType: FileTypes.URL})
const service = builder.addConsumerParameter(numberParameter)
```

We can use the `ServiceBuilder` in combination with the `ConsumerParameterBuilder` to add the parameter to the service metadata.
The supported types of parameters are `'text'`, `'number'`, `'boolean'` and `'select'`.

<Callout type='tip'>
It is also possible to attach consumer parameters to algorithm metadata via [`setAlgorithm()`](/docs/api/assetbuilder/setAlgorithm) to gain access to user input at algorithm runtime.
</Callout>
